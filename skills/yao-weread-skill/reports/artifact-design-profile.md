# 产物设计画像

## 产物类型

交互式 HTML 报告，兼具长文档阅读节奏和仪表盘级证据密度。

## 目标读者

希望复盘近两年阅读行为、个人知识兴趣和笔记习惯的账号所有者。

## 视觉标准

Kami 风格中文报告：暖纸质感、墨蓝强调、编辑化层级、紧凑证据块，不使用装饰性 AI 仪表盘风格。

## 密度标准

用摘要卡片提高扫读效率，再将 20 个以上图表拆入不同主题章节。凡是图表能更清楚表达的模式，不应只依赖文字说明。
图表之外必须有一组内容画像：顶部诗性总结负责“概括这个人”，画像金句负责“点亮首屏”，20 条划线清单负责“用原句支撑画像”。

## 复核清单

- 首屏能说明范围、口径和关键数字。
- 每个图表标题都指向被衡量的阅读行为，而不是泛泛的指标名。
- 图表说明解释源字段或回退口径。
- 矩形树图和热力图在 1024px、桌面宽度和移动宽度下都不能出现明显右侧空白或文字被卡片裁切。
- 画像金句必须来自画像划线清单第一条，划线清单要标注来源书籍。
- 窄屏下布局仍然可读。
- 不出现 API key 或原始密钥。
