# Evidence Map

This internal research map records the evidence buckets used while producing the tutorial package.

| Bucket | Role in Tutorial Design | Use |
|---|---|---|
| User theme and prior draft | Defines the core thesis: high-quality meta-skills come from standing on true giants' shoulders. | Used as the tutorial's central argument, then rewritten into public-facing instructional prose. |
| Agent engineering references | Supplies principles for simple workflows, project memory, context layering, and iteration. | Converted into chapter-level design rules and checklists. |
| Skill and workflow examples | Shows how reusable capabilities can be organized around real tasks. | Used to shape the practice-oriented sections and validation mindset. |
| Tutorial and documentation methodology | Helps distinguish tutorials from reference documentation and ensure beginner-friendly progression. | Used to keep each chapter focused on one concrete learning task. |
| Simplicity and product-planning principles | Helps control complexity and select world-class benchmarks. | Used in the benchmark-selection and iterative-refinement chapters. |

Public outputs must not expose internal source IDs, local file paths, or provenance language. Sources are used to improve judgment and structure, not to make the final tutorial read like a research memo.
