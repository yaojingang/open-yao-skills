# 教程大纲：如何写出高质量元Skill

## 教程定位

面向刚开始写 Agent Skill、元Skill 或可复用 AI 工作流的人。读完后，学习者应该能把一个模糊想法转成一套可研究、可拆解、可验证、可迭代的 Skill 设计流程。

## 开头

从一个常见误区开始：很多人写 Skill 的第一反应是“我来写一段更强的提示词”，但高质量元Skill真正要解决的是“如何把世界级经验压缩进一个可复用流程”。

## 模块 1：先换一个问题

- 核心问题：不是“我怎么写”，而是“谁已经做对了？”
- 学习目标：理解为什么高质量 Skill 需要参考扫描。
- 图示：孤立想法到巨人网络。

## 模块 2：怎么找到真正的巨人

- 核心问题：参考对象如何选择？
- 学习目标：学会用相关性、成熟度、可迁移性、可验证性筛选标杆。
- 图示：标杆筛选矩阵。

## 模块 3：拆解标杆，不抄表面功能

- 核心问题：要拆哪些层？
- 学习目标：掌握结构、流程、交互、记忆、评估、迭代六个维度。
- 图示：六层拆解栈。

## 模块 4：吸收 Agent 工程思想

- 核心问题：CLAUDE.md、上下文管理、任务编排如何进入 Skill？
- 学习目标：把外部 Agent 体系转译成自己的 Skill 设计原则。
- 图示：工程思想转译图。

## 模块 5：用商业策划找世界级参照物

- 核心问题：一个 idea 如何避免自嗨？
- 学习目标：建立“想法 -> 世界级参照 -> 可迁移机制 -> 本地化设计”的路径。
- 图示：对标路径图。

## 模块 6：用奥卡姆剃刀持续迭代

- 核心问题：怎么让 Skill 越迭代越清晰？
- 学习目标：用必要性问题、真实反馈和验证门槛控制复杂度。
- 图示：简化迭代循环。

## 练习

选择一个你想写的 Skill，用本教程的六步方法做一次小型设计。

## 结尾

总结底层逻辑：找到真正的巨人，拆解他们的方法，吸收他们的长处，再用自己的场景持续迭代。
